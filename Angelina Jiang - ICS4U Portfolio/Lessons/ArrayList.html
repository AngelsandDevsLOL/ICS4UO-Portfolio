ArrayLists are similar to arrays. They can store multiple elements of the same type in a contiguous block of memory and can store multiple values. ArrayLists, however, do not have a limited size. It allows the user to add and remove elements. However, ArrayLists are much slower. The ArrayLists comes from the java.util.List interface.
ArrayList<datatype> name = new ArrayList<datatype>(); // More recommended
ArrayList<datatype> name = new ArrayList<>();
Methods

ArrayLists have different methods than arrays.
To access a value in the ArrayList: aList.get(index);
To change a value: aList.set(index, value)
To add a new value: aList.add(index, value) or aList.add(value) to add at the end
To find the index of: aList.indexOf(object)
To find the length of the ArrayList: aList.size()
To remove a value: aList.remove(index) or aList.remove(Object). However, if the object is an integer, then it will remove the index.

Getting the index is the same as an array. The first element starts at 0.
ArrayLists has their own sorting algorithm, called Collections.sort(aList);. However, in order to use Collections.sort() java.util.Collections needs to be imported.
Because ArrayLists are slower than arrays due to the resizing, it takes more time. Therefore, it is more recommended to use arrays when the set size is known, but arraylists are always a capable secondary option.
ArrayLists can contain int, double, String, and even other Objects! Furthermore, even arrays of ArrayLists can be created!
ArrayList<Integer>[] ar = new ArrayList<>[4];
