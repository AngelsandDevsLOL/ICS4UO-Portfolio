Object Oriented Programming is a style of programming when the user creates their own object.
An object is an instance of a class and a class is a blueprint for creating objects with a particular data structure. An object normally has:

  • An identity
  • A state/characteristics -> variables (Example: hairColour:String)
  • A behaviour -> methods (Example: eat():void)
The constructor is a special method which has no return value and the same name as the class
public class Pair { /** class declaration */
    private int xCoordinate; /** instance variables*/
    private int yCoordinate;
    String value = "Hello World";

    /**
     * Pair constructor
     *
     * @param x The x-coordinate
     * @param y The y-coordinate
     */
    public Pair (int x, int y) {
        xCoordinate = x;
        yCoordinate = y;
    }
}

      A class creates an object using a constructor. They are often used with values in their parameters or brackets. The constructor is within the class declaration. If there is no constructor, the object will be created but will have values null. There can be multiple constructors in a class as long as their parameters are different.

    Instance variables, variables declared in the class but outside of any constructors, methods, or blocks, are normally initialized in the constructor unless they start off with a default set value.

The keyword private means that the variable or method cannot be accessed outside of the class. For example, if Pair p was declared and initialized in another class, I would not be able to access p's y value through p.y

Set and Get Methods

  • Also known as accessor and mutate methods respectively

  • Methods in the class that can access or mutate the values of the class
/** * Get xCoordinate method
*
* @return xCoordinate
*/
public int getxCoordinate() {
    return xCoordinate;
}
/**
* Get xCoordinate method
*
* @param xCoordinate set the object's xCoordinate to value of xCoordinate

public void setxCoordinate(int xCoordinate) {
    this.xCoordinate = xCoordinate;
}

UML's

UML stands for Unified Modeling Language. It represents a class with the name, variables, and methods.


An example of a UML is on the left. The name of the class is House. The variables have a "-" symbol to represent that this variable is private. Variables in UML are in the form
-variableName: dataType


Next, we have the constructor with the parameters. For each variable in the parameter, we need to put the name of the variable and the datatype, separated by a colon. For the methods, the "+" symbol states that this method is public. It is in the format
+methodName(parameter with variables): returnDatatype
Inheritance
  • A class, the subclass or is-a class, can inherit all public methods from one and only one other class, the parent class or has-a.
  • A subclass can also override its parent’s methods by having a method with the same name. The methods also need the same parameters in order to have the same signature. The return type does not matter
  • Local variables are variables declared and initialized within a method. Once the method ends, the local variables stop existing.
  • In the constructor of the subclass, there must be super() in the first line of the constructor in order to call the other constructor. Parameters are allowed in super, which declares the parent’s constructor
  • Can also call the overridden methods from the subclass to the parent class by putting super.methodName(with parameters)
  • Private methods are not inherited
  • Subclass cannot directly access private variables

Encapsulation: the process of hiding variables from code in other objects and classes from the outside
Abstraction: the process of hiding unwanted information
public class User {
    private String username;
    private String password;
    public User(String u, String p) {
        username = u;
        password = p;
    }
}
Here, we can see examples of encapsulation and abstraction. We do not want to reveal the password and make it public to other users. This is an example of encapsulation.

We also do not want to make the username public, otherwise it will take up unwanted space. This is an example of abstraction.
public class Pet {
    private String name;
    public Pet (String n) {
        name = n;
    }
    public String toString() {
        return "name: " + name;
    }
}
public class Dog extends Pet {
    private String noise;
    public Dog(String n, String name) {
        super(name);
        noise = n;
    }
}
Here, we can see that the Dog class implements the Pet class. The Pet class is the parent class. This means that the Dog class is the child class. This means that now, the Dog class has the variable name, and can use the toString method. However, it does not inherit the variable, since it is private.

So, if we put Dog d = new Dog("arf!", "Coco");
d.toString(); will return name: Coco
But what if we added a toString() method in the Dog class?
public toString() {
    return "Dog " + super.toString();
}
This is an example of overloading. Overloading is when you use the parent's method equivalent of the method you are in. Overriding is when your parent has the same method, but the child method does not use it. The difference is that overloading uses the parent's method and overriding does not.
Overloading and Overriding are both examples of Polymorphism. Polymorphism is when the correct method is called for an object. So, for example, if we want to declare and initialize Pet p = new Dog("Arf!", "Coco"), we can choose whether we want to print name: Coco or Dog name: Coco through polymorphism.
Notice that Pet p = new Dog("Arf!", "Coco") will not print an error. It will access the Dog constructor. Variable p will be able to access all of Pet's methods. If any of Dog's methods overlap with Pet's methods, then it will go to the Dog's methods. However, if we do Dog d = new Pet(""), it will print out an error since Dog cannot be initialized as a Pet.
However, what if we have Pet p = new Dog("Arf!", "Coco") and we want to access Dog's methods if Pet does not contain those methods? Then, we need to use downcasting. Downcasting is when we cast p to Dog. p is currently declared as a Pet, but since it has been initialized as a Dog, we are allowed to downcast it.
It would look like this:
(Dog p).dogMethod()
We need to put (Dog p) before calling the method, otherwise it would not work.
Abstract class
  • public abstract class
  • can’t create objects
  • Can have constructors, which can be called from constructors of subclasses
  • Abstract classes cannot be instantiated. Method is instantiated in other sub classes
  • If a subclass does not provide an implementation code for all abstract methods, it must be declared abstract
  • Can have abstract methods
  • A class with no abstract methods is called concrete
Interfaces
  • public interface
  • Similar to an abstract class
  • All methods are abstract
  • Does not have any constructors
  • Public abstract does not need to be written because all the methods are public abstract
  • Can implements several interfaces

'this' keyword

  • Can be used as a variable
    • If you were specifying which variable, between the instance variables in the current class and the variables, passed through the parameter or from another class
  • Can be used as a method
    • If you were specifying which method
  • Can be used as an argument
    • it is passing a reference to the current object.
