Arrays

Arrays are blocks or lists that can hold multiple data in the same location. Each array can contain one datatype. Each value in the array is called an element. Furthermore, the size of the array cannot be changed. When the array is declared and initialized, the values will be set to null unless it is a primitive type.

Here is how to initialize an array:
int[] name = new int[size];
Object[] name = new Object[size];

Here is how to change the value in an element:
name[index] = 0;
Here is how to access the value in an element:
int x = name[index];
Arrays are important since they can make code shorter and save time.
int sum = 0;
sum += score0;
sum += score1;
...
sum += score99;

If we already have the values that we want in the array, we can initialize the array like this: int[] values = {1, 2, 3};
Notice that int values[] also works for declaration, but is not proper java syntax.
public int[] method(int[] arr) {
    arr[0] = 0;
    return arr;
}
When passing arrays in the parameter, it is the same as declaring the array.

When returning the array, putting the name of the array would suffice.

Notice that the example code above sets the first element to 0 and returns the array. Arrays are not primitive data types, meaning that when they pass through a method and are changed, the change stays. So, once the method is done, the first element of the array will stay as 0. Thus, returning the array is useless

2D Arrays

2D Arrays is when you have a 1D array, what we have been talking about, and the datatype in your array is an array of all the same datatype.

2D Arrays look like this. In order to access, let's say the 8, we would write arr[1][2]. Remember that the index starts with 0. We do this since we find the row, then the column. For 1D arrays, we only needed to find the column since the row would always be 0.
However, using a table or a chart makes it easier to understand how 2D arrays work! Notice that the array is extended vertically, and it is much easier to read arr[1][2].
1: int[][] ar = new int[4][3]; // 4 rows, 3 columns
2: ar[0][0] = 1;
3: ar[3][2] = 1; // the element at the bottom right
4: int x = ar[0][2];
5: ar = {{1, 2, 3}{4, 5}};
Line 1 creates a rectangular array with a 4 by 3 table.
Line 5 creates a ragged array. A ragged array is when it is weirdly shaped. Now, ar has 2 rows, a maximum of 3 columns, but ar[1][2] does not exist.
To the left, we have a visual representation of the ragged array in Line 5.

The main problem with arrays is that if we want to insert a number, then it will be hard. This is because it will involve creating a new array with a new size and copying the previous array, as well as shifting some elements over.
Using arr.length on a 1D array will give you the number of columns. Using arr.length on a 2D array will give you the number of rows. To get the number of columns for any rows, you can do arr[row].length to get the column.
