Did you know? Understanding Maps is the key to success
Difficulty: Extraordinarily Easy
Sets are like lists, but they cannot contain duplicates.
Maps is like username and password. They are like arrays, where you can get the element through the index. However, they do not have a set-size and the 'index', or key, does not need to be an integer. They cannot contain duplicate keys. If the map has a key and value, then the new value will override the old one.

HashSet follows the Set interface.
TreeSet also follows the Set interface. However, TreeSets sort the values by ascending order.

HashMap follows the Maps interface.
TreeMap also follows the Maps interface. However, TreeMaps sort the key by ascending order.
HashMaps can have both null key and value, but TreeMaps can only have a null key.
Note: Any type of objects can be stored in any sets or maps, but they must have a compareTo() method for any Trees

How to declare and initialize the following objects:
    HashMap<Integer, Integer> hMap = new HashMap<>();
    TreeMap<Integer, Integer> tMap = new TreeMap<>();
    HashSet<Integer> hSet = new HashSet<>();
    TreeSet<Integer> tSet = new TreeSet<>();


The built-in methods, ceilingKey(int) and floorKey(int) can get the key of the key greater than or less than. For example, tMap.ceilingKey(2); will look for the next key greater than 2. If no such key exists, it will return null. The same thing happens with tMap.floorKey(2) except it will look for the next key smaller than 2.

Queue - A list but when you push an element, it goes to the back. When you pop, or take out an element, it can only take out the first element that you’ve put in

Stack - A list but when you push an element, it goes to the back. When you pop, or take out an element, it can only take out the first element that you’ve put in

Deque - A list but when you push an element, it goes to the back. When you pop, or take out an element, it can only take out the first element that you’ve put in

Priority Queue - a queue but the largest element will be moved to the front when an element is popped. Notice that this is time-efficient since it does not sort the entire queue.
