Difficulty: Easy
An algorithm used a lot in competitive programming. The idea is to store the sum from the very beginning of an array
This is useful since it makes it easier to find the sum between any subarray.
A PSA is an array where the sum is stored of all the elements in the original array. For example, a 1-D prefix sum array may look like this:
Side note: A subarray is an array made of any consecutive elements in an array.
int[] originalArr = {1, 2, 3, 4, 5};
int[] psa = {1, 3, 6, 10, 15};
Example Problem: Given an array of size n and q queries, find the sum of numbers between indexes x and y or each query.
The naive solution would be a for loop between x and y, storing the sum and returning.
A faster solution would be calculating the prefix sum array, storing it, and then returning (psa[y] - psa[x - 1])
Notice that the second element is psa[x - 1] because we still want the xth element.
If x - 1 exists and x is the first element, then we normally want to start the array, letting the first element be 0 and x, the first number, to start at index 1. Therefore, the psa[] would actually look like this: int[] psa = {0, 1, 3, 6, 10, 15}
Pseudo code:
for loop through 1 to originalArr.length inclusive
    set the value of psa to the last number, and then added to the original number
for loop through 0 to q
    print the value of psa[y] - psa[x - 1];
for (int i = 1; i <= originalArr.length; i++) {
    psa[i] = psa[i - 1] + originalArr[i - 1];
}
for (int i = 0; i < q; i++) {
    int x = sc.nextInt(), y = sc.nextInt();
    System.out.println(psa[y] - psa[x - 1]);
}

Notice that any sum arrays can be used. A suffix sum array, looping starting from the back to the front, and calculating the sum is useful with a prefix array. A prefix LCM or GCD array can always be handy in some situations. It's important to be flexible and be able to modify algorithms.
